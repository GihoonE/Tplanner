# 개발 기록 · 2026-04-12

## DB · Prisma

- `Session` → `Student` 외래키 `onDelete: SetNull` → **`Cascade`** 로 변경 (학생 삭제 시 해당 수업·숙제 연쇄 삭제)
- 마이그레이션: `20260412162122_session_student_delete_cascade`
- **고아 수업 정리** (예전 SetNull 잔여 등): `npm run db:cleanup-orphan-sessions` — 기본은 미리보기, `--execute` / `--execute --null` 로 실제 삭제

## API

| 경로                 | 메서드             | 설명                                                                         |
| -------------------- | ------------------ | ---------------------------------------------------------------------------- |
| `/api/students/[id]` | GET, PATCH, DELETE | 단건 조회(sessions+homework 포함), 수정(필드 부분 반영·숫자 필드 포함), 삭제 |
| `/api/students`      | GET, POST          | 기존과 동일 (목록·추가)                                                      |

## 페이지 · 컴포넌트

- **학생 관리** (`/students`)
  - **+ 학생 추가** → `AddStudentModal` (POST `/api/students`, 목록·선택 반영)
  - 목록 **변경** → `EditStudentModal` (PATCH/DELETE `/api/students/[id]`)
  - 테이블: 학교·과목·최근 수업 왼쪽 정렬, **수업 시작 월** 열 추가, 열 비율·셀/테이블 가로 스크롤
  - **학생 / 최근 수업 / 상태** 열: 오름·내림 정렬(▲▼)
- **EditStudentModal**: 삭제 확인 문구를 CASCADE 동작에 맞게 수정

## 스크립트

| 명령                                                     | 설명                                                      |
| -------------------------------------------------------- | --------------------------------------------------------- |
| `npm run db:cleanup-orphan-sessions`                     | 고아 수업 건수 미리보기                                   |
| `npm run db:cleanup-orphan-sessions -- --execute`        | 존재하지 않는 학생 id를 참조하는 수업만 삭제              |
| `npm run db:cleanup-orphan-sessions -- --execute --null` | 위 + `studentId` NULL 수업 전부 삭제(초안 수업 포함 주의) |

## 유의사항

- `npm run` 뒤에 스크립트 인자 넘길 때는 **`--`** 뒤에 붙이기 (`-- --execute`)
- `studentId` NULL 수업은 캘린더 초안과 구분 불가 — `--null` 은 정말 비울 때만 사용
